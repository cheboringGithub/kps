# Планирование следующего этапа — программа зала

Это ретроспективный + проектный скилл: смотришь на **весь** пройденный блок (все 10 тренировок), а не последние записи, и предлагаешь **новый блок** (следующий мезоцикл, недели 5-8 и далее). Точечные правки внутри уже написанного блока — это `gym-adjust-program`. Содержание/оборудование конкретных упражнений — по правилам `gym-exercise-review`.

Если вердикт 🔴 — дождись правки `CLAUDE.md`, прежде чем продолжать. Советы из вердикта используй в разделе 4 при формулировке вопросов и в разделе 5 при проектировании блока.

## 2. Проанализировать весовые показатели по каждому упражнению за весь блок

В отличие от `gym-analyze-results` (тренд последних тренировок), здесь нужна **полная кривая по блоку**: для каждого `exercise_id`, встречавшегося в `workouts.ts`, построй последовательность вес×повторы от тренировки 1 до последней пройденной.

- Посчитай суммарный прирост веса/повторов от старта блока к концу (в кг и в % от стартового рабочего веса)
- Определи, какие упражнения росли по double progression как задумано, какие застряли, какие пользователь стабильно не дотягивал до плановых цифр
- Сопоставь с RPE: рост при RPE ≤ «Норм» — есть запас; рост только при RPE «Тяжело»/«Предел» — блок был на пределе адаптации
- Отдельно — колено: любой `knee_pain >= 1` за блок, в связи с каким упражнением, разрешилось ли к концу блока

Прочитай прошлые `gym_analysis_reports.content` целиком — что уже корректировали внутри блока и почему, не предлагай то же самое как новое.

## 3. Определить исходную точку для следующего блока

По итогам раздела 2 — для каждого упражнения, которое продолжится в следующем блоке, определи стартовый рабочий вес (обычно последний пройденный рабочий вес блока, а не пиковый — это начало нового мезоцикла, не продолжение той же прогрессии без паузы). Отметь, какие упражнения выросли настолько, что диапазон повторов/вес пора менять структурно (например, пора сместить рабочий диапазон повторов на более силовой или более объёмный, в зависимости от цели следующего блока).

## 4. Задать пользователю правильные вопросы — обязательный шаг

`gym_entries` фиксирует только вес/повторы/RPE/боль в колене/комментарий — не фиксирует цели, ощущения по конкретным мышцам или изменения условий. Прежде чем проектировать следующий блок, задай вопросы через `AskUserQuestion` **одним вызовом**, с конкретными вариантами:

- **Статус колена у ортопеда** — тот же шлюзовой вопрос, что и в домашней программе (снят ли диагноз/ограничение, назначен ли новый осмотр). Определяет, можно ли возвращать присед/выпады/жим ногами с полной амплитудой.
- **Изменение условий зала**: тот же набор тренажёров, доступ к новому оборудованию, изменилась ли частота (всё ещё 3×/неделю, 60–90 мин)
- **Приоритет следующего блока** — дай конкретные варианты, не спрашивай абстрактно: (а) продолжать тот же баланс сила+гипертрофия, (б) сместить в сторону большего объёма/гипертрофии, (в) сместить в сторону меньшего числа повторов и большего веса (сила), (г) добавить акцент на конкретную зону, где по разделу 2 виден недостаток (например, спина/тяговые, если жимовые росли быстрее)
- **Субъективные ощущения по конкретным упражнениям**, которых нет в `comment` за весь блок — например, было ли что-то дискомфортно кроме колена (плечо в жиме, поясница в RDL)
- Если раздел 2 показал, что пользователь стабильно не дотягивал до плановых цифр в конкретном упражнении — спроси, было ли это ощущение технической сложности, недостатка времени на тренировку или осознанного решения не форсировать

## 5. Предложить план следующего блока — сначала план, потом код

```
# Предложение: следующий блок зала (недели [N]-[N+3]) · [дата]

## Что показал пройденный блок
[весовой прогресс по ключевым упражнениям, что застряло, что с коленом]

## Ответы, повлиявшие на план
[кратко из раздела 4]

## Предлагаемая структура
- Сплит: [сохраняется / меняется, и почему]
- Стартовые веса: [из раздела 3]
- Диапазоны повторов/фокус: [сила/гипертрофия/смешанный]
- Что меняется в упражнениях (если меняется) — по правилам gym-exercise-review
```

Покажи как предложение и дождись подтверждения — это новый 4-недельный блок, ошибка здесь дороже точечной правки.

## 6. После подтверждения — реализация

1. Если нужны новые/заменённые упражнения — следуй `gym-exercise-review` (обязательные `target`/`equipment`, философия выбора тренажёра, чек-лист по колену)
2. Допиши блок в `src/data/gym/workouts.ts` (следующие `week`, double progression от стартовых весов из раздела 3)
3. Закоммить и запушить:

```bash
cd /путь/к/проекту
git add src/data/gym/workouts.ts src/data/gym/exercises.ts
git commit -m "gym-phase: [краткое описание нового блока и почему именно такой]"
git push origin main
```

Версионирование service worker не требуется — сборка через vite-plugin-pwa с автоматическим кэш-бастингом.

4. Доложи пользователю: что показал анализ по блоку, какие ответы повлияли на план, что добавлено и почему.
