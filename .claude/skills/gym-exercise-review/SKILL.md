# Ревью упражнений — программа зала

Этот скилл — про содержимое самих упражнений в `src/data/gym/exercises.ts` (техника, целевая мышца, оборудование). Если задача — поменять состав тренировок по неделям, это `gym-adjust-program`, не этот скилл.

Если причина правки — чисто редакторская (уточнить `equipment`/`target`, добавить `cues`), `health-check` не нужен. Если причина клиническая (новый симптом, снятие/подтверждение ограничения по колену и т.п.) — сначала вызови `health-check`, чтобы убедиться, что ограничение зафиксировано в `CLAUDE.md`, и только потом редактируй упражнение.

## 1. Модель данных (`src/data/gym/types.ts`)

Каждый `GymExercise`: `name`, `equipment`, `target`, `how`, `cues`, `note`.

- **`equipment`** — конкретный тренажёр/инвентарь, не общее слово. «Тренажёр» само по себе не годится — напиши какой именно (рычажный/Хаммер, блочный с широкой рукоятью, тренажёр для сгибания ног лёжа, тренажёр для отведения/приведения бедра сидя и т.д.). В зале много похожих тренажёров — читатель должен понимать, к какому именно подойти.
- **`target`** — обязательное поле, не пропускай. Одна строка: (а) целевая мышечная группа и (б) механика движения, включая положение тела (сидя/стоя/лёжа) и для тяговых — направление («откуда куда»). Пример: «Широчайшие мышцы спины, бицепс — тяга сидя от верхнего блока вниз к верху груди».
- **`how`** — HTML-строка (рендерится через `dangerouslySetInnerHTML` в `GymExerciseCard.tsx`, как и в домашней программе). Исходное положение отдельным предложением, затем выполнение, темп/дыхание.
- **`cues`** — 2-3 подсказки по технике (не по целевой мышце — это уже в `target`): стоп-точка, частая ошибка, на что не отвлекаться.
- **`note`** — safety-caveat (ограничение по колену), приоритет стороны или обоснование замены оборудования. `null`, если сказать нечего.

## 2. Философия выбора оборудования

Не «всегда тренажёр» — конкретное правило:

- **Меняй свободный вес на тренажёр**, когда (а) движению нужна страховка партнёра, которой нет (жим штанги лёжа/стоя над лицом/головой), или (б) кабельный/свободный вариант ощутимо менее контролируем, чем тренажёрный аналог на ту же мышцу (пример: стоячее кабельное отведение ноги в сторону заменено на сидячий тренажёр отведения/приведения бедра).
- **Не трогай** штангу/гантели/трос там, где они уже безопасны и хорошо контролируемы: RDL, тяги в наклоне, сгибания на бицепс, антиротация на блоке, фейс-пул — не заменяй их тренажёром без причины.
- Раз тренажёров в зале много и они похожи — при любом упражнении указывай ровно тот, что нужен, и его настройку (положение валиков, угол скамьи), а не общее название категории.

## 3. Ограничения, которые нельзя нарушать

Полные правила — в `CLAUDE.md` (раздел «Зал: правила формирования программы»), здесь чек-лист:

- **Колено** (тендинит связки надколенника слева, до контрольного осмотра ортопеда): никакой осевой нагрузки через согнутое стоя колено — под запретом присед любой, выпады и вариации, walking lunges, степ-апы, глубокий жим ногами, RDL на одной ноге, тяга с выраженным сгибанием колена стоя. Разрешено: разгибание/сгибание ног в тренажёре (контролируемая амплитуда), жим ногами с ограниченной амплитудой, отведение/приведение бедра, ягодичный мост двумя ногами.
- **Левая средняя ягодичная и хамстринг — приоритет объёма**, как и в домашней программе: в односторонних безопасных упражнениях левая сторона выполняется первой и получает больше объёма — `note` должен явно это объяснять.
- **Верх тела в зале — настоящая силовая работа на массу/силу**, не постуральная коррекция (та уже покрыта домашней программой) — не занижай веса/прогрессию здесь так, как это оправдано для домашних постуральных упражнений.
- **Не дублируй домашний кор** (планка/боковая планка/мёртвый жук/птица-собака) — кор в зале должен быть нагруженным вариантом (антиротация на блоке и т.п.), либо не добавляй его вовсе в этот день.

## 4. Процедура ревью/добавления/улучшения упражнения

1. Прочитай `CLAUDE.md` (раздел «Зал») и память — ограничение по колену снимается только по факту осмотра ортопеда, не по времени, проверь текущий статус, а не полагайся на память из старой сессии.
2. `grep` по `src/data/gym/workouts.ts` на id упражнения — используется в скольких из 10 тренировок / 4 недель, с какими sets/reps/weight. Переименование или удаление id требует обновить ВСЕ вхождения.
3. Если удаляешь упражнение, которое использовалось в нескольких неделях с разной прогрессией (double progression: сначала повторы растут, потом вес) — не сливай его числа в оставшееся соседнее упражнение бездумно. Прогрессия каждого упражнения считается независимо от его собственной истории, а не от того, что стояло рядом.
4. Напиши/поправь поля по требованиям раздела 1, выбор оборудования — по разделу 2.
5. `npx tsc --noEmit`.
6. Визуально проверь в Browser pane: разверни карточку, убедись что `target`/`how`/`cues`/`note` реально отрендерились. Клики по заголовку карточки иногда промахиваются мимо координат (особенно после скролла) — если ничего не изменилось, проверь через `document.querySelector` в `javascript_tool`, а не только скриншотом/`get_page_text`.
7. Доложи пользователю: что изменено, что убрано, чем заменено и почему.

## 5. Частые ошибки (из практики)

- Оставить `equipment`/`target` расплывчатыми («тренажёр», «жим ногами») без указания конкретной модели и положения тела — именно то, что нужно исправлять этим скиллом.
- Переименовать id без грепа всех вхождений в `workouts.ts` — упражнение молча пропадает из недели.
- При удалении упражнения слить его вес/повторы в оставшееся соседнее без пересчёта — прогрессия должна оставаться логичной для конкретного упражнения, а не быть механическим переносом чужих цифр.
