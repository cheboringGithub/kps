# Ревью упражнений — домашняя программа КПС

Этот скилл — про содержимое самих упражнений в `src/data/exercises.ts` (техника, целевой эффект, безопасность). Если задача — поменять, какие упражнения стоят в каком дне программы, это `adjust-program`, не этот скилл.

Если причина правки — чисто редакторская (уточнить формулировку, добавить `cues`, расширить `how`), `health-check` не нужен. Если причина клиническая (новый симптом, снятие/подтверждение ограничения по колену и т.п.) — сначала вызови `health-check`, чтобы убедиться, что ограничение зафиксировано в `CLAUDE.md`, и только потом редактируй упражнение.

## 1. Модель данных (`src/data/types.ts`)

Каждый `Exercise`: `name`, `time`, `type`, `timer`, `how`, `cues`, `note`, опционально `repsOnly`.

- **`how`** — HTML-строка (рендерится через `dangerouslySetInnerHTML` в `ExerciseCard.tsx`). Разрешены только `<strong>`, `<br>`, `<em>`, кружки `①②③`. Обязательно: исходное положение отдельным предложением, затем пронумерованные шаги для многоэтапных движений, темп/повторы в конце.
- **`cues`** — массив ровно 2-3 коротких технических подсказок. Тип `string[]`, не опционален — пустой массив недопустим. Подсказки НЕ пересказывают `how`, а добавляют: стоп-точку, частую ошибку, дыхание, что не должно двигаться.
- **`note`** — только когда есть клиническая причина (почему это упражнение важно именно для этого паттерна) или caveat безопасности. Не заполняй искусственно, если сказать нечего — `null`.

## 2. Ограничения, которые нельзя нарушать

Полные правила — в `CLAUDE.md`, здесь только чек-лист для быстрой проверки перед правкой:

- **Колено (тендинит связки надколенника слева, действует до контрольного осмотра ортопеда):** никакой осевой нагрузки через согнутое стоя колено. Запрещённый список: `il_lunge`, `il_lunge_adv`, `squat_sumo`, `strength_reverse_lunge`, `strength_split_squat`, `strength_single_rdl`, `strength_hip_thrust`, `glute_bridge_single`, `strength_side_steps` — не назначай их в новые дни, но не удаляй из каталога (это история уже пройденных дней). Хамстринг-растяжки безопасны при этом диагнозе.
- **Стопа — супинация, не пронация:** единственное упражнение на стопу — `foot_eversion`. Не создавай `foot_peroneal`/`foot_short`/`foot_outer`/`foot_heel_raise` заново — они уже были удалены как обоснованные через опровергнутый паттерн пронации.
- **Левый КПС/ягодичная/КМП — всегда приоритет:** в асимметричных упражнениях левая сторона держится дольше / получает больше повторов, и `note` объясняет почему (не просто «для симметрии»).
- **Острая боль в пояснице по утрам:** не давай скручивания при активной боли; наклоны вперёд стоя — только когда утренней боли нет, иначе `cues` должны указывать лежачую альтернативу.
- **Не дублируй стимул:** прежде чем добавлять новое упражнение, проверь, не покрывает ли его уже существующее в той же плоскости движения (пример уже закреплённого правила: `hip_abduction_3way` перекрывает `glute_clamshell`/`glute_side_lying` — не ставь их в один день; `glute_side_kicks` не дублирует, это другой паттерн).
- **Прогрессия силовых** (`strength_*`, `core_*`, мосты/тяги): должна расти от фазы к фазе — повторы, удержание или сопротивление. Не повторяй одни и те же цифры в `days.ts` несколько фаз подряд без причины.

## 3. Процедура ревью/добавления/улучшения упражнения

1. Прочитай актуальный `CLAUDE.md` (раздел «Контекст пользователя» и «Правила работы») и память — ограничения меняются во времени (диагноз колена уточнялся), не полагайся на то, что помнишь из предыдущей сессии.
2. `grep` по `src/data/days.ts` на id упражнения — где и как часто используется. Если переименовываешь/удаляешь id, обнови все вхождения.
3. Напиши/поправь поля по требованиям раздела 1.
4. Сверь с чек-листом раздела 2 — особенно колено и дублирование стимула.
5. `npx tsc --noEmit`.
6. Визуально проверь в Browser pane: разверни карточку, убедись что `how`/`cues`/`note` реально отрендерились (не только компилируются) — `dangerouslySetInnerHTML` молча проглатывает битую разметку, поэтому смотри на итоговый вид, а не только на исходник. Клики по заголовку карточки иногда промахиваются мимо координат — если после клика ничего не появилось, проверь через `document.querySelector` в `javascript_tool`, а не только скриншотом.
7. Доложи пользователю: что изменено, что добавлено, почему — без этого он не поймёт логику асимметрии/замены (см. правило «Обновление программы» в `CLAUDE.md`).

## 4. Частые ошибки (из практики)

- Переименование id без грепа всех вхождений в `days.ts` — упражнение молча пропадает из дня.
- Оставлять `cues` дублирующими `how` слово в слово — подсказки должны добавлять новую информацию, не пересказывать.
- Забыть, что запрещённые по колену упражнения нельзя удалять из каталога совсем — они остаются как история уже пройденных дней, ограничение действует только на **новые** назначения.
