## 2. Правила коррекции

Работаешь с конкретным паттерном: **левый КПС опущен и сжат, левая средняя ягодичная заторможена, компенсация через левое колено и правый квадрицепс**.

### Если поясница утром ≥ 2 держится >4 записей подряд
- Вставь в ближайший незавершённый день дополнительный `kps_decomp` на место одного упражнения типа `secondary`
- Убери из следующих 3 дней любые упражнения стоя (`hamstring_standing`, `il_standing`, `it_standing`)
- Не трогай упражнения лёжа

### Если kps_feeling = 1 держится >6 записей
- Замени один день с `figure4` на `kps_figure4_floor` (поза голубя мощнее)
- Добавь `kps_90_90` в следующий день если его там нет
- Если уже есть оба — добавь второй цикл `kps_traction` (поставь `kps_traction` дважды в списке одного дня)

### Если left_knee = 2 держится >3 записей
- Добавь `it_side_lying` и `it_standing` в ближайшие 2 дня
- Убери `hamstring_standing` пока knee > 1 (заменяй на `hamstring_seated`)
- Убери `squat_sumo` если он есть в ближайших 5 днях

### Если sitting_symmetry = 'да' держится >8 записей
- Увеличь асимметрию: добавь повтор дня специально под среднюю ягодичную
- Вставь дополнительный день с составом: `warmup_knee`, `glute_clamshell`, `glute_side_lying`, `glute_bridge_single`, `kps_traction`, `kps_clock`, `savasana`

### Если kps_feeling = 3 стабильно >5 записей И back_pain ≤ 1
- Программа идёт впереди плана → ускорь переход на следующую фазу
- Сдвинь начало фазы 3 на 2 дня раньше (убери 2 последних дня фазы 2, вставь 2 первых дня фазы 3 вместо них)

### Если всё в норме (back_pain ≤ 1, kps ≥ 2, knee = 0, symmetry ≠ 'да')
- Не меняй ничего
- Сообщи пользователю: программа работает, коррекция не нужна, пуш не нужен

## 3. Как редактировать days.ts

- Корректируй только **незавершённые дни** — спроси у пользователя: "Какой последний день ты выполнил?"
- Не трогай фазу 1 если пользователь уже прошёл её
- Изменения вноси точечно: минимум правок, максимум эффекта
- Правила раздела 2 используют только уже существующие в каталоге упражнения. Если под конкретный случай в каталоге нет подходящего упражнения (нужна замена/добавление, которого нет в списке выше) — вызови скилл `exercise-review`, прежде чем писать новую запись в `src/data/exercises.ts`: он задаёт модель данных (`how`/`cues`/`note`) и чек-лист ограничений (колено, стопа, дублирование стимула)

## 4. После изменений — коммит и пуш в main

1. Запиши изменения в `src/data/days.ts`
2. Обнови версию service worker в `sw.js`: найди строку `kps-v` и увеличь номер на 1
3. Добавь запись в конец `training-analysis.md` (что изменено, почему)
4. Закоммить и запушить:

```bash
cd /путь/к/проекту  # используй реальный путь из workspace
git add src/data/days.ts sw.js training-analysis.md
git commit -m "adjust: [краткое описание что изменено и почему]"
git push origin main
```

Используй в commit message конкретику: какой показатель триггернул, что изменено. Например:
`adjust: back_pain≥2 x5 → убраны стоячие упражнения дни 8-10, добавлен kps_decomp`

5. После успешного пуша скажи пользователю что изменено и что CI сейчас собирает обновление.
